Evals 2026-07-08

Separating signal from noise in coding evaluations

出典: OpenAI

読む価値: SWE-Bench Pro を監査した結果、約 30% の task に問題があると示しました。ベンチマークの見た目の伸びと、実際の能力は別物だと分かります。

開発者への示唆: coding eval は「ある」だけでは不十分で、タスク品質の監査と失敗原因の分解が必要です。モデル比較より先に、評価データの健全性を疑うべき局面があります。

試すなら: まず自分の評価セットを 10〜20 件だけ監査し、壊れたタスクや曖昧な合格条件を洗い出す。外部ベンチマークの結果と内部の失敗パターンを突き合わせる。

Infra 2026-07-08

Enterprise-managed OpenTelemetry export for VS Code and CLI

出典: GitHub Changelog

読む価値: Copilot Chat と Copilot CLI の telemetry を、管理設定で組織側の collector に強制できます。`OTEL_*` を各開発者が手で触らなくていいのは実務で大きいです。

開発者への示唆: agent 実運用では、モデルの性能よりも先に観測と統制が求められます。managed exporter headers や content capture の制御は、監査・セキュリティ設計に直結します。

試すなら: まず collector の置き場所と export 形式を決め、managed-settings.json でどこまで固定できるかを確認する。セキュリティ要件がある環境では先にこの経路を通す。

Metrics 2026-07-07

Add review cycles and time to adoption phases in the usage API

出典: GitHub Changelog

読む価値: Copilot usage metrics API に、レビュー初回までの時間と review cycle 数が増えました。AI adoption phase ごとにレビュー速度を追えるのがポイントです。

開発者への示唆: AI 導入の効果を merge 数だけで見ると雑です。レビュー待ちが減ったか、反復回数が減ったかまで見ると、組織のボトルネックが見えます。

試すなら: enterprise / org レポートを 1 回引き、各 phase の `avg_pull_requests_minutes_to_review` と `avg_pull_requests_review_cycles` を比較する。採用が進んだチームほどレビューが軽くなっているかを確認する。

Agents 2026-07-07

Codex as agent provider and agentic enhancements in JetBrains IDEs

出典: GitHub Changelog

読む価値: JetBrains IDE で Codex が public preview の agent provider になり、Hooks、MCP 管理、カスタムモデル、CLI 承認設定までまとまって扱えるようになりました。

開発者への示唆: agent の選択はモデルだけでなく、IDE・CLI・MCP・承認フローの組み合わせで決まります。複数 provider を前提にすると、導入と切り替えの設計が変わります。

試すなら: 1 つの JetBrains プロジェクトで Codex provider を有効化し、MCP サーバーの管理と承認設定を含めて普段の作業に乗るか確かめる。
