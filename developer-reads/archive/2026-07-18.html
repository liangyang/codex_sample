Infra Evals

Repository-level GitHub Copilot usage metrics generally available

出典: GitHub Changelog / 公開日: 2026-07-17 JST

読む価値: Copilot usage metrics REST API が repository-level の pull request activity を返すようになり、coding agent と code review の動きを repo 単位で見られます。

開発者への示唆: Copilot の導入評価が org 全体の数字から、どの repo が実際に使っているかという運用判断へ下がってきました。

試すなら: repos-1-day を定期取得して、PR 量が多い repo と Copilot 活動の偏りを見てください。

Copilot code review: Customization and configurability improvements

出典: GitHub Changelog / 公開日: 2026-07-17 JST

Agents Workflow

Building scalable AI agents with modular prompt transpilation

出典: Google Developers Blog / 公開日: 2026-07-16 JST

読む価値: 単一の巨大 prompt を分割し、skill fragments を build 時に検証する考え方です。失敗を runtime ではなく CI に寄せられます。

Agents RAG

Expanding Choice in Gemini Enterprise Agent Platform: Introducing Grounding with Parallel Web Search

出典: Google Developers Blog / 公開日: 2026-07-16 JST

読む価値: Parallel Web Systems の grounding provider が Gemini Enterprise Agent Platform に入り、リアルタイム検索結果を citations 付きで参照できるようになりました。

開発者への示唆: grounding は「検索するか」ではなく、「どの provider を信頼し、どう証跡を残すか」の設計問題になります。

試すなら: 事実依存の agent で grounded response の citation を監査ログに残し、非 grounding 経路との差を比較してください。

テーマ別メモ

  • metrics as control plane: 利用分析が org 単位から repo と app の粒度に下がり、運用判断に使いやすくなっています。

  • grounding as provider choice: grounding は検索有無ではなく、証跡と信頼境界をどう設計するかの問題です。
