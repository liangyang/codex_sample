Safety 2026-07-10

CodeQL 2.26.0 adds Kotlin 2.4.0 support and AI prompt injection detection

出典: GitHub Changelog

読む価値: Kotlin 2.4.0 対応に加え、JavaScript / TypeScript で system prompt injection 検出クエリが追加されました。OpenAI / Anthropic / Google GenAI SDK の入力流入もより広く見られます。

開発者への示唆: AI 機能をプロダクトに入れるほど、プロンプト注入や SDK 経由の入力流入を静的解析で拾う価値が上がります。安全性はモデル側だけでなくコード側でも守る必要があります。

試すなら: まず `js/system-prompt-injection` の検出結果を確認し、AI SDK の untrusted input 経路を見直す。CodeQL の更新を CI に織り込む。

API 2026-07-10

Per-user states for multi-user budgets in the REST API

出典: GitHub Changelog

読む価値: 複数ユーザーの budget 消費を 1 つの REST API で一覧できるようになり、限界に近い人を素早く見つけられます。大規模組織での AI 利用統制がしやすくなります。

開発者への示唆: AI 利用の制御は、機能の導入だけでなく予算の可視化と介入が必要です。大きな組織ほど、個別 API 呼び出しの自前スクリプトから脱した方がよくなります。

試すなら: multi-user budget の一覧を 1 回引き、使用率でフィルタして上位利用者を洗う。個人 override がある場合の表示も確認する。

Agents 2026-07-10

GitHub Mobile: Improved filters and sorting for Copilot sessions

出典: GitHub Changelog

読む価値: Copilot sessions をモバイルで探しやすくなり、Active / Status / Repository / Type / Agent で絞り込めます。セッションが増えても、必要なものをすぐ拾えます。

開発者への示唆: agent 利用が増えるほど、モバイルでのセッション triage や active / needs-attention の把握が実務になります。長い作業を回す人ほど、一覧性の改善が効きます。

試すなら: まずセッション数が多い状態で、Active と needs-attention を切り替えて探しやすさを確認する。Repository と Agent の組み合わせで絞れるか試す。
