Infra Evals

GitHub Code Quality is now generally available

出典: GitHub Changelog / 公開日: 2026-07-20 JST

読む価値: Code Quality が Enterprise Cloud と Team で GA になり、CodeQL の決定論的解析に AI-assisted detection と Copilot Autofix が重なりました。org ダッシュボード、coverage 指標、ruleset の quality gate、API もそろっています。

開発者への示唆: AI で速く書けることと、AI で安全に通すことが同じ管理面に乗ってきました。レビュー補助ではなく、CI とガバナンスの一部として扱う必要があります。

試すなら: 既存の lint / test / CodeQL と 1 つの repo で比較し、coverage gate を小さく有効化して triage 負荷を見てください。

Infra Billing

AI credit pools for cost centers in the billing UI

出典: GitHub Changelog / 公開日: 2026-07-20 JST

読む価値: cost center の AI credit pool を billing UI で直接管理できるようになりました。割当ライセンス数から上限が自動計算され、到達時の動作も UI で選べます。

開発者への示唆: AI の利用制御が API 専用ではなく、運用者が日常的に触る画面まで降りてきました。チームごとの上限と overage の設計が現実的になります。

試すなら: 1 つの cost center に pool と budget を併用し、上限到達時に stop するか追加課金に流すかを実地で確認してください。

Infra Billing

Copilot users can now see AI credits used per billing cycle

出典: GitHub Changelog / 公開日: 2026-07-20 JST

読む価値: 個人の Copilot 画面で、budget がなくても今の billing cycle に何クレジット使ったかが見えるようになりました。budget がある場合は消費済み量と上限の両方を確認できます。

開発者への示唆: 利用量の把握が「管理者だけの指標」から「使う本人のフィードバック」に降りてきました。プロンプトや agent ワークフローの調整がしやすくなります。

試すなら: 代表的な作業を 1 週間追い、どの流れが credits を押し上げているかを見てください。

Safety Evals

Safety and alignment in an era of long-horizon models

出典: OpenAI / 公開日: 2026-07-20 JST

読む価値: 長時間動くモデルは難しい問題を解ける一方で、意図しない行動を取る機会も増えます。OpenAI は実運用中の失敗から、trajectory-level monitoring と新しい eval を追加しました。

開発者への示唆: autonomous agent は最終出力だけ見ても足りません。途中経路の監視、停止条件、ロールバック手順まで含めて設計する必要があります。

試すなら: 1 つの長時間ワークフローに対して trajectory を記録し、逸脱時の中断条件を先に決めてください。
