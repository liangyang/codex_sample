API 2026-07-09

GPT-5.6: Frontier intelligence that scales with your ambition

出典: OpenAI

読む価値: GPT-5.6 は Sol / Terra / Luna の3系統で一般提供され、長時間タスクとコスト効率の両方を引き上げます。複数エージェントを並列に走らせる `ultra` の設計も示されました。

試すなら: 同じプロンプトを `Sol` / `Terra` / `Luna` で回して、速度・品質・コストを比較する。長いタスクは `ultra` 相当の分割が効くか確認する。

Evals 2026-07-08

Separating signal from noise in coding evaluations

出典: OpenAI

読む価値: SWE-Bench Pro を詳細監査した結果、約 30% の task に問題があると示しました。見かけのスコア改善と、実際の能力が一致しないことを再確認できます。

開発者への示唆: coding eval は結果だけでは足りず、曖昧な合格条件、過剰に厳しいテスト、低カバレッジを疑うべきです。モデル比較の前に評価データの健全性を確認する必要があります。

試すなら: 自分の評価セットを少数でも監査し、壊れたタスクや誤った採点条件を洗い出す。外部ベンチと内部失敗ログを突き合わせて、何が本当に改善したのかを切り分ける。

Infra 2026-07-08

Enterprise-managed OpenTelemetry export for VS Code and CLI

出典: GitHub Changelog

読む価値: Copilot Chat と Copilot CLI の telemetry を、管理設定で組織側の collector に固定できます。各開発者が `OTEL_*` を手で揃える必要がありません。

開発者への示唆: agent 導入の初期課題は、性能より先に可観測性と情報統制です。managed exporter headers や content capture の制御は、監査とセキュリティ設計に直結します。

試すなら: collector の置き場所と export 形式を先に決め、managed-settings.json でどこまで固定できるかを確認する。VS Code と Copilot CLI を同じ telemetry 方針で揃える。

API 2026-07-10

Add review cycles and time to adoption phases in the usage API

出典: GitHub Changelog

読む価値: Copilot usage metrics API に、レビュー初回までの時間と review cycle 数が増えました。AI adoption phase ごとにレビュー速度を追えるのがポイントです。

開発者への示唆: AI 導入の効果を merge 数だけで見ると雑です。レビュー待ちが減ったか、反復回数が減ったかまで見ると、組織のボトルネックが見えます。

試すなら: enterprise / org レポートを 1 回引き、各 phase の `avg_pull_requests_minutes_to_review` と `avg_pull_requests_review_cycles` を比較する。採用が進んだチームほどレビューが軽くなっているかを確認する。

Safety 2026-07-10

CodeQL 2.26.0 adds Kotlin 2.4.0 support and AI prompt injection detection

出典: GitHub Changelog

読む価値: CodeQL 2.26.0 で Kotlin 2.4.0 対応が入り、JavaScript / TypeScript では system prompt injection の検出クエリが追加されました。

開発者への示唆: AI 機能をプロダクトに入れるほど、プロンプト注入や SDK 経由の入力流入を静的解析で拾う価値が上がります。安全性はモデル側だけでなくコード側でも守る必要があります。

試すなら: まず `js/system-prompt-injection` の検出結果を確認し、OpenAI / Anthropic / Google GenAI SDK の入力経路を見直す。CodeQL の更新を CI に織り込む。
