Safety Coding Agents

Code scanning shows AI security detections on pull requests

出典: GitHub Changelog / 公開日: 2026-07-14 JST

読む価値: CodeQL ではまだ届かない言語やフレームワークまで、AI-powered security detections を PR 上で直接見られるようになりました。マージ前に見つける設計が明確です。

開発者への示唆: これは代替ではなく補完です。既存の CodeQL、Dependabot、secret scanning の隙間を埋める軽量レイヤーとして使うのが筋です。

試すなら: 1 つのアクティブ PR で有効化し、CodeQL の結果と並べて差分と取りこぼしを確認してください。

Safety Coding Agents

Security reviews now available in the GitHub Copilot app

出典: GitHub Changelog / 公開日: 2026-07-14 JST

読む価値: `/security-review` で、作業中の変更に対する脆弱性レビューを Copilot app 内から即時に回せます。重要なのは「後でスキャン」ではなく「作業中に直す」ことです。

開発者への示唆: 安全性チェックが専用ツールから日常の編集フローへ入ってきています。コードレビューの前提が少し変わります。

試すなら: ブランチ上の変更を開いて `/security-review` を実行し、レビュー観点の抜けを洗ってください。

Safety Evals

GPT-Red: Unlocking Self-Improvement for Robustness

出典: OpenAI / 公開日: 2026-07-15 JST

読む価値: 自動レッドチームを使って脆弱性を見つけ、その攻撃データで GPT-5.6 を adversarial に鍛える話です。安全性改善を手作業から反復ループへ持ち込んでいます。

開発者への示唆: prompt injection への対策は、ルール追加よりも「攻撃生成」「検証」「再学習」の回路を作れるかで差が出ます。

試すなら: 自分の agent に対して、メール・Web・tool output を使う prompt injection の再現セットを作り、同じ traces を継続的に回してください。

Evals Inference

Lessons From the Leaderboard: What 5,000+ Kagglers Taught Us About Improving AI Reasoning

出典: NVIDIA Technical Blog / 公開日: 2026-07-14 JST

読む価値: reasoning 改善の要点を、verifiable な trace、token budget、再利用できる memory、tool で作る training data、task-type ごとの評価に分解しています。

開発者への示唆: 単一スコアの平均値より、失敗型ごとの分解と再現性のほうが実装に効きます。agent や coding workflow にそのまま当てはまります。

試すなら: eval を task type で分割し、trace の中間ステップを検証してから、圧縮後の入力でも同じ精度が出るか比べてください。
