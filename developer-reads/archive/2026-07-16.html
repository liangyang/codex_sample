Agents Infra

Expanding Choice in Gemini Enterprise Agent Platform: Introducing Grounding with Parallel Web Search

出典: Google Developers Blog / 公開日: 2026-07-16 JST

読む価値: Gemini Enterprise Agent Platform に Parallel Web Systems の grounding provider が入り、リアルタイム検索結果に citations を付けて参照できるようになりました。事実確認が必要な agent に直結します。

開発者への示唆: web grounding は「検索するかどうか」ではなく「どの provider を信頼し、どう証跡を残すか」の設計問題になっています。

試すなら: KYC、情報収集、社内調査のような事例で、grounded result の citation を downstream の監査ログに残す流れを作ってみてください。

Agents Workflow

Building scalable AI agents with modular prompt transpilation

出典: Google Developers Blog / 公開日: 2026-07-16 JST

読む価値: 単一の巨大 prompt を分割し、skill files を transpile して build 時に検証する考え方です。agent の失敗を runtime ではなく build pipeline で潰す発想が実務向きです。

開発者への示唆: agent の信頼性は prompt の美しさではなく、依存関係の明示、静的検証、CI 統合で決まります。

試すなら: 既存の system prompt を instructions、policy、tooling、format などに分割し、欠落検出を CI に入れる最小構成を作ってみてください。

Evolving Spec-Driven Development: Conductor Now Supports Antigravity

出典: Google Developers Blog / 公開日: 2026-07-16 JST

読む価値: spec-driven development を Gemini CLI extension から plugin に進化させ、skills、rules、MCP servers、hooks をまとめて扱える形にしています。開発フローを会話に寄せつつ、成果物は version-controlled に保てます。

Agents SDK

Agent Skills for Python Is Now Released

出典: Microsoft Agent Framework / 公開日: 2026-07-15 JST

読む価値: instructions、reference material、scripts をまとめた reusable skill を、stable な production API で扱えるようになりました。必要な時だけ load するので context を圧迫しません。

開発者への示唆: agent の能力は巨大 prompt に閉じ込めず、チーム単位で再利用できる skill として配るのが自然です。

試すなら: よく使う domain knowledge を 1 つ skill 化して、Python agent が必要時だけ読み込む構成に置き換えてみてください。

Safety Infra

Improvements to secret scanning and public monitoring

出典: GitHub Changelog / 公開日: 2026-07-15 JST

読む価値: secret scanning の対象が増え、webhook には `secret_category` が入り、public monitoring には要約カードが追加されました。漏えい検知を運用しやすい形に寄せています。

開発者への示唆: security signal を UI だけで終わらせず、分類された webhook と監査フローに流せるかが実装差になります。

試すなら: 既存の secret scanning webhook handler で `secret_category` を分岐し、generic と provider-specific の triage を分けてください。
